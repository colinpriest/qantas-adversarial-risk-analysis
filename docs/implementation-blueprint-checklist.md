### 1.2 LLM Elicitation Prompts

**Board pipeline** (`board_utility_quantification.py`):

The system prompt establishes a simulated Qantas boardroom with 8 named directors, legal/regulatory context (three liability channels: personal s180, two-strikes spill, corporate class actions), and the September 2023 crisis background. The response format requests: action probabilities (summing to 1.00), 10 factor ratings (1-5 Likert, randomised order), and free-form commentary.

When the CEO is present, the prompt activates four cognitive bias mechanisms: (1) anchoring on 100% peer departure base rate, (2) bandwagon effect via 5 named ASX peers, (3) loss aversion via regulatory/legal consequences of inaction, (4) counter-bias deactivation (naming retention biases: escalation of commitment, status quo bias, groupthink, over-optimism, hyperbolic discounting). These biases are calibrated to match the empirical observation that 100% of comparable ASX 100 ESG crises resulted in CEO departure.

**ASA pipeline** (`asa_utility_quantification.py`):

Simulates ASA (Australian Shareholders' Association) deliberations over strike recommendations at the A2 node. Uses context + interaction phi decomposition: context terms fire equally for both actions (cancel in delta-EU), interaction terms fire only for `rec_strike` (drive the decision).

Scenarios are constructed by crossing state dimensions to achieve full coverage of the utility parameter space, with additional scenarios for identification of specific parameters (e.g., `w_ceo_accountability`, `w_review_post_removal`).

### 1.4 Sampling Loop

The solver iterates over belief draws, evaluating the game tree for each:

```
For each checkpoint C in {Cpre, C0, C1, C2, C3}:
  beliefs = BeliefBundle(checkpoint_path)

  For each scenario s in {ceo_stayed, ceo_resigned}:
    state = base_state.for_scenario(s)

    For each candidate action a at initial node:
      values = []

      For draw_i in range(n_draws):        # Typically 100-500
        belief_draw = beliefs.get_draw(draw_i)

        # Epistemic draws (once per belief draw, held fixed across MC):
        governance_effect ~ U(floor, ceiling)     # D1 action effectiveness
        crisis_floor ~ Beta(50, 150)              # Vote structural floor
        outcome_probs ~ Dirichlet(38, 160, 1)     # Review rating probs

        # Tree recursion with nested MC sampling:
        value = tree.evaluate(next_node, history, state, draw_i, mode, rng)
        #   At chance nodes: average over n_vote_samples=50 or n_review_samples=20
        #   At focal decisions: maximise over feasible actions
        #   At opponent decisions: weight by predictive distribution
        #     Predictive: K=200 opponent draws x R=20 forward rollouts
        #   At terminals: compute utility for focal actor

        values.append(value)

      EU[a] = mean(values)

    optimal_action = argmax(EU)
    Pr_scenario = predict_d0_ceo(focal, checkpoint)  # ARA prediction of CEO pre-game
```

### 1.5 Parameter Estimation

**Board utility weights** (Bayesian ordinal probit):

| Stage | Process | Input | Output |
|-------|---------|-------|--------|
| 1 | Scenario generation | governance_spec.xlsx | ~1,800 scenario state vectors |
| 2 | LLM elicitation | Scenarios + system prompt | Likert ratings (1-5) per action x n_draws |
| 3 | Preprocessing | Raw Likert data | Long-format CSV + summary statistics |
| 4 | Stan estimation | Likert long-format + phi matrix | Posterior draws (w_raw_1..w_raw_K) via ordinal probit |
| 5 | Behavioural diagnostics | Likert data + factor ratings | Anchoring, order effects, consistency checks |
| 6 | Validation | Posterior draws + game tree | Interactive HTML dashboard with EU decomposition |

The ordinal probit model (`models/ordinal_utility.stan`):

```
y[i] ~ OrderedProbit(phi[i] . w, cutpoints)
```

where `phi[i]` is the scenario-specific feature vector and `w` are the utility weights with lognormal priors. Cutpoint reparameterisation ensures identifiability.

**ASA utility weights** (constrained optimisation + method-of-moments):

| Stage | Process | Input | Output |
|-------|---------|-------|--------|
| 1 | Range elicitation | LLM boardroom simulation | Usual [lo, hi] and rare [lo, hi] for each node's P(strike) |
| 2 | Gap elicitation | LLM scenario comparison | Pairwise probability gaps between adjacent nodes |
| 3 | Monotonicity solving | Targets + ranges + ranking | MC-optimised P(strike) per node via 2-stage L-BFGS-B + Nelder-Mead |
| 4 | Weight calibration | Optimised probabilities | TruncNormal utility weight distributions (mu, sigma per weight) |
| 4a | Beta fitting | P(strike) + elicitation SD | Method-of-moments Beta(alpha, beta) per node for opponent modelling |
| 6 | Engine integration | Beta params + point estimates | `asa_a2_calibration.json` consumed by predictive.py and game_tree.py |

### 1.6 Tree Engine

**Node dispatch** (`engine/tree.py: TreeEvaluator.value()`):

```
value(node, history, state, draw_i, mode, rng):
  match node_type:
    Terminal → compute utility for focal actor via utility_{board,asa,ceo}()

    Decision (focal) → max over feasible actions:
      for action in feasible_actions(node, state):
        child_value = value(next_node, history + {node: action}, state.apply(action))
      return max(child_values)

    Decision (opponent) → weighted sum via ARA predictive:
      pred_dist = predictive.predict(node, history, state, draw_i, focal, mode, level)
      return sum(prob * value(next_node, ..., action) for action, prob in pred_dist)

    Chance (V) → Monte Carlo over vote samples:
      for _ in range(n_vote_samples):
        vote = chance_models.sample_vote(draw_i, beliefs, history, state, rng, ...)
        values.append(value(next_node, history + {V: vote}, state.apply(vote)))
      return mean(values)

    Chance (R) → Monte Carlo over review samples:
      for _ in range(n_review_samples):
        review = chance_models.sample_review(draw_i, beliefs, history, state, rng, ...)
        values.append(value(next_node, history + {R: review}, state.apply(review)))
      return mean(values)
```

**Opponent modelling** (`engine/predictive.py: PredictiveDistribution.predict()`):

```
predict(node, history, state, draw_i, focal, mode, level):
  owner = node_owner(node)                    # The opponent
  feasible = feasible_actions(node, state)
  counts = {a: 0 for a in feasible}

  for k in range(K):                          # K=200 opponent parameter draws
    theta_k = param_sampler.sample(focal, owner, rng)  # From focal's priors

    eu_per_action = {}
    for action in feasible:
      rollout_values = []
      for r in range(R):                      # R=20 forward simulations
        terminal = simulate_forward(next_node, history + {node: action}, ...)
        rollout_values.append(utility(owner, terminal, theta_k))
      eu_per_action[action] = mean(rollout_values)

    best = argmax(eu_per_action)
    counts[best] += 1

  # Laplace smoothing: (count + 1) / (K + |feasible|)
  return {a: (counts[a] + 1) / (K + len(feasible)) for a in feasible}
```

**Game tree node order:**

```
D0_ceo → D1 → A2 → V → D4 → D_rev → [R → D4_post → D_rev_post →] Terminal
                                        ↑                              ↑
                                  if review commissioned         if negative + CEO present
```
